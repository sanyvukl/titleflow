# TitleFlow

TitleFlow is a secure digital vehicle title and lien processing system built as a full-stack portfolio project.

The application simulates a real-world workflow where dealers submit vehicle title applications, DMV clerks review and approve or reject applications, lenders manage liens, and admins manage users and system records.

## Project Goals

This project is designed to teach and demonstrate:

- Java 21
- Spring Boot
- Spring Security
- REST API development
- PostgreSQL
- JPA/Hibernate
- React
- TypeScript
- Material UI
- Authentication and role-based access control
- Secure document upload
- Audit logging
- RabbitMQ messaging
- Docker
- Testing
- AWS deployment basics

## Main User Roles

- Admin
- Dealer
- DMV Clerk
- Lender

## Core Workflow

1. Dealer creates a title application.
2. Dealer enters vehicle, buyer, seller, and lien information.
3. Dealer uploads required documents.
4. Dealer submits the application.
5. DMV clerk reviews the application.
6. DMV clerk approves, rejects, or requests more information.
7. Lender manages lien records.
8. System records audit logs and notifications.
